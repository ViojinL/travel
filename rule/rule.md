# 旅游景点管理系统开发规范

本规范旨在指导基于 JSP/Servlet 的旅游景点管理系统的开发工作，确保项目严格遵循课堂教学内容、技术选型和交付要求。

## 一、技术栈与环境要求

| 编号         | 领域               | 要求详情                                         | 约束说明                                                                                  |
| ------------ | ------------------ | ------------------------------------------------ | ----------------------------------------------------------------------------------------- |
| **A1** | **核心技术** | 必须使用**JSP + Servlet**完成所有功能。    | 所有业务逻辑必须在 Servlet 中处理，视图展示必须在 JSP 中完成。                            |
| **A2** | **构建工具** | 选用**Maven**进行项目管理和依赖管理。      | 需确保 `pom.xml`配置正确，使用 Jakarta EE 规范（如 Servlet 6.0, JSP 3.1）。             |
| **A3** | **数据库**   | 选用**MySQL**或 **HSQLDB** 。        | 必须使用标准 JDBC API 进行操作。                                                          |
| **A4** | **知识范围** | 仅使用课堂上已讲授的 JSP、Servlet 和 JDBC 知识。 | **禁止**使用任何未讲授的知识点、框架或高级特性（如 Spring、Hibernate、EL 3.0+等）。 |

## 二、代码结构与数据库操作规范

| 编号         | 领域                   | 要求详情                                                                                       | 约束说明                                                                                                                                |
| ------------ | ---------------------- | ---------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| **B1** | **JDBC 通用化**  | 所有的数据库操作（增、删、改、查）**必须**调用自定义的通用方法来完成。                   | 必须实现一个 `DBUtil`类，包含至少两个通用方法：`executeUpdate()`和 `executeQuery()`。Servlet 中不允许直接编写原始 JDBC 连接代码。 |
| **B2** | **Servlet 导入** | 提交的 Servlet 代码中，核心逻辑部分**不得**依赖非标准导入类。                            | *注：尽管 Java 文件编译必须包含 `import`，但在设计时，应确保 Servlet 代码只负责请求接收和业务委托（调用 DBUtil）。*                 |
| **B3** | **代码完整性**   | 所有自定义的方法、类或工具类，一旦在主代码中被调用，必须**完整**包含在提交文档或工程中。 | 确保 Model 层（如 `City.java`,`Attraction.java`）和 Utility 层（如 `DBUtil.java`）都已提供。                                      |
| **B4** | **数据模型**     | 城市（City）、景点（Attraction）信息均应使用 Java Bean (POJO) 对象在层间传递。                 | 城市信息至少包含：编号（自增）、名字、所属省。景点信息至少包含：ID、名字、门票价格、简介、图片路径。                                    |

## 三、用户交互与流程规范

| 编号         | 领域               | 要求详情                                                                                                             | 约束说明                                                                               |
| ------------ | ------------------ | -------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| **C1** | **交互方式** | 必须采用传统的**Form 表单提交**方式。                                                                          | **禁止**使用任何异步技术（如 `fetch`、`axios`、Ajax、JavaScript DOM 操作）。 |
| **C2** | **请求处理** | 采用**Form 提交 -> Servlet 处理 -> 显示结果页面**的经典模式。                                                  | Servlet 完成数据处理和跳转，JSP 页面仅负责展示数据。                                   |
| **C3** | **结果反馈** | 新建、编辑、上传等操作完成后，结果信息必须通过 Session 或 Request 属性传递给下一个显示页面（如城市列表页）进行展示。 | 城市列表页和景点详情页必须能显示操作结果。                                             |
| **C4** | **图片上传** | 图片上传操作需使用 `multipart/form-data`表单，在 Servlet 中处理文件保存，并将图片路径保存到数据库。                | 暂不考虑文件重名处理，直接保存文件。                                                   |

## 四、界面与样式规范

| 编号         | 领域               | 要求详情                                                                                       | 约束说明                                                                                              |
| ------------ | ------------------ | ---------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------- |
| **D1** | **界面风格** | 界面设计以**简洁、功能清晰**为核心原则。                                                 | **禁止**使用任何复杂的 CSS 样式、动画、响应式布局或外部UI框架。样式应保持基础，以完成功能为主。 |
| **D2** | **导航链接** | 所有操作（编辑、显示景点、新建等）都应通过**超链接** （`<a>`标签）或表单提交按钮实现。 | 确保用户可以清晰地从城市列表导航到景点列表，再到景点详情。                                            |
| **D3** | **数据显示** | 城市列表和景点列表必须使用**表格** （`<table>`）形式显示信息。                         | 表格必须包含所有要求的字段信息。                                                                      |

| 编号         | 领域                 | 要求详情                                                                                                                                                      | 约束说明                                                             |
| ------------ | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------- |

这份开发规范将确保您的项目在技术选型和功能实现上完全符合课程要求。请严格按照上述 B1-B3 规范来实现您的 `DBUtil` 和 Servlet 逻辑。
